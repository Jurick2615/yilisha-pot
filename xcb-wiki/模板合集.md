# 📋 xcb 知识库模板合集

> 按场景分类，你只需打开对应模板 **填空** 即可。

---

## ⚡ 模板 1：快速记录（一句话小知识）

适合：临时记的小技巧，随手留

```markdown
---
title: 短标题
tags: [标签1, 标签2]
---

一句话记录。比如：
> 合同序号是 @ 时，需要启用才会生效。
```

---

## 🧩 模板 2：实体/系统说明

适合：介绍一个 **系统、模块、席位、网关、数据库**

```markdown
---
title: xxx 系统
type: entity
tags: [xcb, 深市, 网关]
---

## 概述
这个系统是干什么的？

## 关键信息
- **IP/端口：** xxx
- **账号：** xxx（如果可公开）
- **关联系统：** [[系统A]]、[[系统B]]

## 配置要点
1. 关键配置项及含义
2. 默认值 / 推荐值

## 注意事项
- 踩过的坑
- 已知限制
```

## 🔧 模板 3：操作流程/升级步骤

适合：**升级、部署、开市、关机** 等标准步骤

```markdown
---
title: xxx 操作流程
type: procedure
tags: [升级, 深市, 网关, mgr]
---

## 前置条件
- [ ] 已完成备份
- [ ] 已通知相关人员
- [ ] 确认当前版本号：xxx

## 操作步骤

### 1. 第一步
```bash
# 命令示例
```

### 2. 第二步
操作说明

### 3. 验证
```bash
# 验证命令 / 预期输出
```

## 回退方案
如果失败怎么办？

## 预期耗时
约 xx 分钟

## 常见坑
- ⚠️ 注意：xxx
```

## ❌ 模板 4：故障处理/FAQ

适合：**报错、异常、客户问题**

```markdown
---
title: xxx 故障
type: faq
tags: [常见故障, xcb]
---

## 现象
客户看到什么报错？

## 原因
为什么会出现？

## 解决方案
### 方案 A：简单处理
1. 步骤 1
2. 步骤 2

### 方案 B：升级处理（如果方案 A 无效）
1. 步骤 1
2. 步骤 2

## 预防措施
如何避免再次出现？
```

## ✅ 模板 5：巡检/检查清单

适合：**每日巡检、开市前检查、周检、月检**

```markdown
---
title: xxx 检查清单
type: checklist
tags: [巡检, 深市]
---

## 检查项

### 1️⃣ 网络/连接
- [ ] 网关连接正常
- [ ] 行情通道正常
- [ ] mgr 状态正常

### 2️⃣ 数据
- [ ] 昨日清算完成
- [ ] 今日参数已加载

### 3️⃣ 日志
- [ ] 无异常报错
- [ ] 待处理告警已清零

## 结果记录
检查人：_____
检查时间：_____
异常项：_____
```
